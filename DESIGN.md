你是一名精通 **Editorial Design（编辑设计）** 与 **前端工程化** 的资深设计师与全栈开发者。请基于以下完整的设计系统规范，为一款**杂志风独立媒体网页应用**输出可落地的设计方案与前端实现代码。

该设计必须**原生融合暗色模式（Dark Mode）**，两种模式需保持高度一致的严肃报刊气质，拒绝简单的"反色逻辑"。

---

## 一、设计哲学

- **气质**：严肃、克制、强排版、纸质印刷感。
- **核心原则**：留白即内容、排版即图像、色彩仅服务阅读层级。
- **双模式统一性**：暗色模式不是"夜间版"，而是"高级深灰印刷版"。两种模式共享完全一致的布局网格、字号阶梯与组件结构，仅通过色彩令牌切换氛围。

---

## 二、双模式色彩系统（Design Tokens）

所有颜色必须使用 CSS 变量实现，禁止硬编码。

| 令牌名 | 亮色模式 | 暗色模式 | 用途与说明 |
|---|---|---|---|
| `--bg-primary` | `#F4F1EA` | `#0A0A0A` | 主背景。暗色禁用纯黑，保留极微弱暖灰质感。 |
| `--bg-secondary` | `#EAE6DD` | `#151515` | 次级背景/卡片底色。用于引用块背景、精选列表悬停。 |
| `--text-primary` | `#111111` | `#F0EDE6` | 主文本。暗色禁用纯白，使用带暖调的纸白。 |
| `--text-secondary` | `#6B665E` | `#8A8580` | 次文本、元信息、时间戳。暗色保持暖灰，避免冷蓝灰。 |
| `--surface-dark` | `#080808` | `#1A1A1A` | 深色专题卡/黑底区块。暗色下比背景略亮，形成层次。 |
| `--surface-invert` | `#080808` | `#F0EDE6` | **反转专题卡**：暗色模式下可偶尔使用浅色卡片（类似杂志插页），文字随背景反转。 |
| `--border` | `#D8D1C4` | `#2A2722` | 分割线、边框、引用线。暗色下保持暖调深灰。 |
| `--accent` | `#111111` | `#D8D1C4` | 强调色：编号、首字下沉、悬停下划线。暗色使用浅暖灰提亮。 |
| `--hover` | `rgba(17,17,17,0.06)` | `rgba(240,237,230,0.06)` | 链接与列表项悬停背景。 |

**色彩禁忌**：
- 暗色模式**绝对禁止**使用 `#000000` 纯黑背景或 `#FFFFFF` 纯白文字。
- 两种模式均**禁止使用任何高饱和度彩色**（如纯红、亮蓝、荧光绿）作为强调色。
- 禁止使用冷色调灰（如 `#334155`、`#64748B`）进入暗色模式。

## 三、字体系统（Typography）

| 层级 | 字体 | 字号 | 字重 | 行高 | 字间距 | 暗色模式特殊处理 |
|---|---|---|---|---|---|---|
| **H1 首屏大标题** | Playfair Display / Georgia | `clamp(3rem, 6vw, 5.5rem)` | 400 | 1.05 | `-0.02em` | 字重保持 400，暗色下衬线体过粗会显得臃肿。 |
| **H2 文章标题** | Playfair Display / Georgia | `2.25rem` | 400 | 1.2 | `-0.01em` | 同上 |
| **H3 小标题** | Playfair Display / Georgia | `1.5rem` | 400 | 1.3 | `0` | — |
| **正文** | Georgia / "Noto Serif SC" / 系统衬线体 | `1.125rem` | 400 | 1.8 | `0.01em` | 暗色下字间距增至 `0.02em`，提升可读性。 |
| **引用块** | Playfair Display Italic | `1.25rem` | 400 | 1.7 | `0` | 文字色使用 `--text-primary`，左侧边框使用 `--accent`。 |
| **元信息/注释** | system-ui / sans-serif | `0.875rem` | 400 | 1.5 | `0.02em` | 字色 `--text-secondary`，无衬线体保证小号字清晰。 |
| **编号/序号** | Playfair Display | `1.5rem` | 600 | 1 | `0` | 暗色下颜色使用 `--accent`。 |

**排版规则**：
- 正文栏宽：`max-width: 38rem`（约 640px），居中，保持最佳阅读行长。
- 首屏大标题可突破栏宽，使用 `max-width: 48rem`，但保持左对齐或居中对齐（根据版面选择，需统一）。
- **严禁使用无衬线字体作为标题**（破坏杂志感）。

## 四、布局系统（Layout）

- **全局留白**：页面左右边距 `clamp(1.5rem, 5vw, 4rem)`。
- **网格**：首屏使用非对称网格（如 12 列中标题占 8 列，元信息占 4 列）；正文区域严格单栏。
- **分割线**：使用 1px solid `--border`，而非阴影。分割线上下留白 `3rem`。
- **专题卡间距**：黑底专题卡与上下内容间距 `4rem`，内部 padding `3rem`。
- **暗色模式布局一致性**：暗色下不改变任何网格比例、栏宽或间距数值。

## 五、组件规范（含双模式样式）

### 1. 大标题（Hero Title）
- 亮色：文字色 `--text-primary`，无背景，下方 1px `--border` 分割线。
- 暗色：完全相同的样式，仅变量自动切换。

### 2. 文章元信息（Meta Bar）
- 包含：作者、日期、阅读时间、分类标签。
- 样式：字号 `0.875rem`，字色 `--text-secondary`，分类标签使用 `1px solid --border` 边框包裹，无背景，圆角 `0`（杂志风拒绝圆角）。
- 暗色：边框与文字色自动适配。

### 3. 引用块（Blockquote）
- 结构：左侧 `2px solid --accent` 边框，左内边距 `1.5rem`，上下 `2rem`。
- 背景：亮色使用 `--bg-secondary`（极淡的暖灰）；暗色使用 `--bg-secondary`（深灰 `#151515`）。
- 文字：Playfair Display Italic，`--text-primary`。
- **禁忌**：禁止使用引号图标（`"` 图形）作为装饰，杂志风靠排版本身说话。

### 4. 编辑精选列表（Editor's List）
- 结构：编号（01, 02, 03...）+ 标题 + 分割线。
- 编号：Playfair Display，`--accent`，字号 `1.25rem`，位于标题左侧。
- 悬停：背景色 `--hover`，过渡 `0.2s`。
- 暗色：编号颜色从黑色变为 `--accent`（浅暖灰），保持清晰但不刺眼。

### 5. 黑底专题卡（Dark Feature Card）
- **亮色模式**：`background: --surface-dark`，`color: #F4F1EA`（固定暖白，不使用变量避免过暗），内部大标题 Playfair Display，`1px solid rgba(244,241,234,0.1)` 的微妙边框可选。
- **暗色模式（默认）**：`background: --surface-dark`（`#1A1A1A`），`color: --text-primary`（`#F0EDE6`）。此时卡片比页面背景 `#0A0A0A` 略亮，形成优雅的层级。
- **暗色模式（反转插页变体）**：允许偶尔使用 `background: --surface-invert`（`#F0EDE6`），`color: #0A0A0A`，制造杂志中"浅色插页"的强对比效果。**此变体需手动标记，不可全局使用。**

## 六、动效与交互规范

- **总体原则**：尽量少用。动效必须服务于阅读节奏，而非装饰。
- **页面加载**：首屏大标题与正文采用 `opacity: 0 → 1` 淡入，`duration: 0.8s`，`ease-out`。
- **滚动显现**：正文图片与次级标题使用 `translateY(20px) → 0` 配合淡入，`duration: 0.6s`，`ease-out`，仅在进入视口时触发一次，禁止重复触发。
- **链接悬停**：下划线动画。默认无下划线，`border-bottom: 1px solid transparent` → `border-color: --accent`，过渡 `0.2s`。
- **模式切换过渡**：
  - 全局 `transition: background-color 0.4s ease, color 0.4s ease, border-color 0.4s ease`。
  - 切换按钮本身：太阳/月亮图标旋转 `180deg` 或淡入淡出切换，`0.3s`。

## 七、暗色模式专项交互规范

1. **切换控件**：
   - 位置：导航栏最右侧。
   - 样式：纯图标按钮，线条风格（`stroke-width: 1.5px`），无背景填充，圆角 `0`。
   - 图标：太阳（亮色模式显示）/ 月亮（暗色模式显示），使用 SVG。
   - 悬停：`background: --hover`。

2. **状态管理**：
   - 默认：`prefers-color-scheme` 媒体查询自动匹配。
   - 用户手动切换后：保存至 `localStorage`，优先级高于系统偏好。
   - **防闪烁（FOUC）**：必须在 `<head>` 内嵌入内联脚本（先于 CSS 加载），读取 `localStorage` 并给 `<html>` 打上 `data-theme="dark"` 属性。

3. **图片适配**：
   - 暗色模式下，正文图片亮度建议通过 CSS 统一微调：`filter: brightness(0.95)`，避免亮色图片在暗色下刺眼。
   - 必须为图片设置 `::after` 伪元素或背景色兜底，防止加载延迟时出现白色占位撕裂暗色氛围。

---

## 八、绝对禁忌（两种模式通用）

- **禁止**使用圆角超过 `2px` 的组件（按钮、卡片、图片）。杂志风需要锐利、干净的边缘。
- **禁止**使用阴影（`box-shadow`）作为层级区分手段。层级仅通过留白、分割线、色彩对比建立。
- **禁止**使用渐变背景（尤其是暗色模式下的蓝紫渐变、霓虹渐变）。
- **禁止**使用 emoji 或卡通图标作为功能标识。
- **禁止**在暗色模式使用高对比度的纯白（`#FFF`）与纯黑（`#000`）组合。
- **禁止**使用无衬线字体（如 Inter、PingFang SC）作为标题字体。
- **禁止**正文使用小于 `1rem` 的字号。
- **禁止**任何弹跳、弹性、摇晃类动效。
