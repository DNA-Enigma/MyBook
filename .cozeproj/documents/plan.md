# 个人综合服务门户 — 实现计划

## 概述

构建一款部署在 Ubuntu 服务器的个人综合服务网页应用。核心功能包含：笔记管理（CRUD + 分类标签）、个人作品展示画廊、个人简介页面、综合资源库（软件/文档/图片媒体/Docker镜像）的上传下载与展示。支持多用户注册登录，管理员与访客分角色权限控制，确保数据安全。

## 技术方案

| 维度 | 选择 | 理由 |
|------|------|------|
| 前端框架 | Next.js 16 (App Router) | 全栈支持，SSR/SSG，API Routes 一体化，部署便捷 |
| UI 组件 | shadcn/ui + Tailwind CSS v4 | 现代化组件库，快速构建，主题一致性 |
| 后端 API | Next.js API Routes | 与前端同仓库，开发效率高，无需单独部署 |
| 数据库 | Supabase (PostgreSQL) | 托管数据库，内置认证，RLS 行级安全 |
| 认证方案 | Supabase Auth | 内置 JWT 认证，支持邮箱/密码注册登录，角色管理 |
| 文件存储 | 对象存储 (S3 兼容) | 大文件上传下载，预签名 URL 安全访问 |
| 部署环境 | Ubuntu + Node.js | 用户明确要求 Ubuntu 服务器部署 |

## 功能模块

### 1. 用户认证与权限
- 注册/登录/登出（邮箱+密码）
- JWT Token 管理
- 角色系统：`admin`（管理员）、`user`（普通用户）、`guest`（访客，未登录可浏览公开内容）
- 权限控制：管理员可增删改所有内容；普通用户可管理自己的笔记；访客仅浏览公开内容

### 2. 笔记管理
- 笔记列表页：卡片式展示，支持搜索、分类筛选、标签过滤
- 笔记详情页：富文本/Markdown 渲染
- 笔记编辑页：创建/编辑笔记，支持标题、内容、分类、标签、公开/私密设置
- 数据结构：`id`, `title`, `content`, `category`, `tags[]`, `is_public`, `author_id`, `created_at`, `updated_at`

### 3. 个人作品展示
- 作品画廊页：网格/瀑布流布局，支持分类（设计/代码/摄影等）
- 作品详情页：大图展示、描述、技术栈标签、外部链接
- 作品编辑：管理员可上传封面图、填写描述（仅管理员可操作）
- 数据结构：`id`, `title`, `description`, `cover_image_url`, `category`, `tech_stack[]`, `external_link`, `is_public`, `created_at`

### 4. 个人简介
- 静态展示页：头像、姓名、简介文案、技能标签、联系方式、社交链接
- 管理员可在后台编辑简介内容
- 数据结构：`id`, `avatar_url`, `name`, `bio`, `skills[]`, `contact`, `social_links`

### 5. 综合资源库
- 资源列表页：分类标签页切换（软件/文档/图片媒体/Docker镜像），支持搜索
- 资源上传：管理员可上传文件，填写名称、描述、分类标签
- 资源下载：通过预签名 URL 安全下载
- Docker镜像：展示镜像名称、版本、拉取命令、描述（纯文本展示，不上传实际镜像文件）
- 数据结构：`id`, `name`, `description`, `file_url`, `file_type`, `file_size`, `category` (software\|document\|media\|docker), `docker_pull_cmd`, `download_count`, `is_public`, `created_at`

## 实施步骤

### 阶段一：原型设计
1. **加载 design-canvas 技能，完成全站原型设计** — 输出首页、笔记页、作品页、简介页、资源库页、登录注册页的高保真 HTML 原型。完成后提示用户验收，确认后进入开发阶段。

### 阶段二：代码开发
2. **初始化项目与配置环境** — `coze init` 创建 Next.js 项目，配置 shadcn/ui 主题，安装 Supabase/Storage SDK，配置 `.env` 环境变量。关键文件：`package.json`, `globals.css`, `.env.local`
3. **搭建数据库 Schema 与认证系统** — 创建 Supabase 表（users, notes, works, profiles, resources），配置 RLS 策略，实现注册/登录/登出 API 和前端 Hook。关键文件：`src/lib/supabase.ts`, `src/app/api/auth/*`
4. **实现用户认证页面与全局布局** — 开发登录/注册页、顶部导航栏（含用户信息/角色展示）、权限路由守卫。关键文件：`src/app/login/page.tsx`, `src/app/register/page.tsx`, `src/components/navbar.tsx`
5. **开发笔记管理模块** — 笔记列表（搜索/筛选）、笔记详情、笔记编辑/创建页，支持 Markdown 渲染和公开/私密权限。关键文件：`src/app/notes/page.tsx`, `src/app/notes/[id]/page.tsx`, `src/app/notes/edit/page.tsx`
6. **开发作品展示与个人简介模块** — 作品画廊（网格布局+分类筛选）、作品详情、个人简介展示页及后台编辑。关键文件：`src/app/works/page.tsx`, `src/app/profile/page.tsx`
7. **开发综合资源库模块** — 资源列表（分类标签页）、资源上传（含文件上传进度）、资源下载、Docker镜像信息展示。关键文件：`src/app/resources/page.tsx`, `src/app/api/upload/route.ts`, `src/app/api/download/route.ts`
8. **权限控制集成与测试验证** — 全站 API 添加权限中间件，前端按钮/操作按角色条件渲染，执行静态检查与接口冒烟测试，确保数据安全。关键文件：`src/middleware.ts`, `src/lib/permissions.ts`

## 数据安全与权限控制

### 数据库层面
- Supabase RLS 策略：所有表启用行级安全
- `notes` 表：公开笔记所有人可读；私密笔记仅作者和管理员可读
- `works` / `profiles` / `resources` 表：所有人可读公开内容；仅管理员可写入/修改
- `users` 表：仅自身可读取自己的敏感信息

### API 层面
- 所有写操作 API 校验 JWT Token 和角色权限
- 非公开资源下载需校验权限
- 文件上传限制文件类型和大小

### 前端层面
- 按用户角色条件渲染操作按钮（编辑/删除/上传）
- 路由级别权限守卫：管理员后台页未授权自动跳转登录
