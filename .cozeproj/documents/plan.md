# 首页改造与资源库上传修复计划

## 概述

改造首页为个人门户风格：左侧固定展示站长个人信息卡片，右侧流式堆叠展示笔记、作品、资源等最新内容。同时修复资源库上传问题，确保 .exe、.deb、.dmg、压缩包、源码等各类文件都能正常上传。

## 技术方案

| 维度 | 选择 | 理由 |
|------|------|------|
| 首页布局 | 左侧边栏(站长信息) + 右侧主内容区 | 用户明确要求左侧放个人信息，右侧堆叠展示内容 |
| 数据获取 | Server Component + Drizzle ORM | 首页需要查询 profiles/notes/works/resources 多表数据 |
| 大文件上传 | S3 对象存储分片上传(>50MB) | Supabase TUS 有 50MB 硬限制 |
| 小文件上传 | Supabase Storage TUS 直传(≤50MB) | 小文件直接走 TUS |
| 文件类型限制 | 无限制(全开放) | 资源库需支持任意软件/代码资源 |

## 功能模块

### 1. 首页个人门户改造

- **左侧站长信息卡片**：头像、姓名、简介、技能标签、社交链接(GitHub/网站/邮箱)、角色标识
- **右侧内容流**：
  - 最新笔记（卡片列表，带分类标签、摘要、作者、日期）
  - 最新作品（卡片列表，带封面图、技术栈标签）
  - 最新资源（卡片列表，带文件类型图标、大小、下载数）
  - 每项展示 3-6 条，点击跳转详情
- **空态处理**：每项有独立空态提示，不显示生硬的"暂无信息"

### 2. 资源库上传修复

- 确保前端 `<input type="file">` 无 `accept` 限制
- 确保 S3 分片上传和 TUS 路径均无扩展名/MIME 白名单
- 确保下载时保留原始文件名
- 文件分类覆盖：软件、安装包(Windows/macOS/Linux)、压缩包、源码、Docker镜像、文档等

## 是否有原型设计

是 — 设计引导已开启，首页涉及 UI 布局改动，需先进行原型设计。

## 实施步骤

### 阶段一：原型设计

1. **加载 design-canvas 技能，完成首页原型设计** — 左侧站长信息 + 右侧内容堆叠布局，使用 skeleton-web.html 骨架，生成 `.cozeproj/prototype/web/home.html`

### 阶段二：代码开发

2. **改造首页为个人门户布局** — `src/app/page.tsx` 左侧站长信息卡片 + 右侧笔记/作品/资源流式展示，查询 profiles/notes/works/resources 表真实数据
3. **修复资源库上传文件类型支持** — 检查 `src/app/resources/page.tsx` 和上传路由，确保无文件类型限制，各种安装包/压缩包/源码均可上传
4. **优化空态展示与下载体验** — 首页和内容页空态使用友好提示；下载路由确保文件名和 Content-Type 正确
5. **运行代码静态检查、接口测试、日志检查** — `pnpm lint` + `pnpm ts-check` + 接口冒烟测试 + 日志健康检查

## 页面规格

##### @nav(web-topbar)
> type: topbar
> platform: web

- @page(/) 首页
- @page(/notes) 笔记
- @page(/works) 作品
- @page(/resources) 资源库
- @page(/profile) 个人中心

##### @page(/) 首页

**核心职责**：展示站长个人信息和最新内容流，作为个人门户入口。
**访问路径**：导航栏 Logo / 直达。
**布局**：
- 顶部：固定导航栏（沿用现有 navbar）
- 主区域：左右分栏布局
  - 左侧（约 1/3）：站长信息卡片 — 头像、姓名、角色徽章、简介、技能标签云、社交链接图标
  - 右侧（约 2/3）：内容流堆叠展示
    - 最新笔记区：横向卡片或列表，展示 3-4 篇最新公开笔记
    - 最新作品区：网格卡片，展示 3-4 个最新公开作品
    - 最新资源区：列表，展示 4-6 个最新公开资源
  - 每个区域有标题 + "查看全部"链接

**状态**：
- 空态：某类内容为空时显示"还没有发布 [笔记/作品/资源]，去创建一篇吧" + 快捷跳转按钮
- 加载态：骨架屏 shimmer 效果

**交互说明**

| 元素 | 动作 | 响应 | 传参 | 备注 |
|------|------|------|------|------|
| 笔记卡片 | 点击 | 跳转 @page(/notes)?note_id | note_id | — |
| 作品卡片 | 点击 | 跳转 @page(/works)?work_id | work_id | — |
| 资源项 | 点击 | 跳转 @page(/resources) | — | 跳转到资源库列表 |
| 查看全部 | 点击 | 跳转对应列表页 | — | — |
| 社交链接图标 | 点击 | 新标签页打开外部链接 | — | GitHub/网站/邮箱等 |
